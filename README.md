# Lean SQL-Gen MVP

A lightweight, browser-based utility that converts natural language business criteria into valid SQL using a "Schema-in-Prompt" strategy via OpenRouter and Claude 3.5 Sonnet.

Unlike complex RAG systems, this tool reads a dedicated `tables.md` Star Schema file and injects it directly into the LLM context, bridging the gap between business logic and database queries with zero infrastructure.

![Lean SQL-Gen generating SQL with syntax highlighting](.gemini/antigravity/brain/21c24290-8c3d-4e28-a1ae-290ed745902e/generated_sql_correct_highlighting_1775669900928.png)

## 📌 Features

- **Simplified NL2SQL:** Convert business questions (e.g., *"Show total revenue by store for Q3"*) into valid SQL.
- **Zero-Infrastructure RAG:** Uses a Markdown file (`tables.md`) as the single source of truth for your DB schema. No ChromaDB or Vector embeddings needed.
- **Premium UI:** Dark-mode glassmorphism interface with custom syntax-highlighted SQL.
- **Docker-ready:** Spin up the entire stack in under 10 seconds.

## 🚀 Tech Stack

- **Backend:** Node.js, Express
- **Frontend:** EJS, Vanilla CSS (Glassmorphism), Vanilla JS
- **AI Provider:** OpenRouter API (Default: `anthropic/claude-sonnet-4`)
- **Deployment:** Docker (Alpine based)

## 📦 Getting Started

### 1. Requirements

- [Node.js](https://nodejs.org/) v18+
- An [OpenRouter](https://openrouter.ai/) API Key

### 2. Setup 

Clone the repository and install dependencies:

```bash
npm install
```

### 3. Environment Variables

Ensure there is an `.env` file in the root directory containing your OpenRouter API key:

```env
OPEN_ROUTER_API="sk-or-v1-..."
```

### 4. Running Locally

To run the application with automatic restarts:

```bash
npm run dev
```

The application will be available at [http://localhost:3000](http://localhost:3000).

## 🛠 Usage & Configuration

### Updating the Schema

The AI generates SQL based *entirely* on the contents of the `tables.md` file located in the root directory. To point the application to your own database:

1. Open `tables.md`.
2. Document your tables, columns, data types, and primary/foreign key relationships.
3. Reload the application UI. The new schema will automatically be injected into future AI prompts.

### Changing the AI Model

By default, the application uses Claude 3.5 Sonnet. You can modify this in `server.js` by changing the `model` property inside the `callOpenRouter` function block.

## 🐳 Docker Deployment

The application is fully containerized for out-of-the-box portability.

```bash
# Build the image
docker build -t lean-sql-gen .

# Run the container
docker run -p 3000:3000 --env-file .env lean-sql-gen
```
